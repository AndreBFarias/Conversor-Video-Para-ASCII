# Para Onde Queremos Ir

**Data:** 2025-12-18

## TL;DR

Transformar Luna em uma assistente pessoal autônoma e completa, com capacidades emocionais, plugins extensíveis e integração profunda com o ambiente desktop. Norte estelar: IA gótica que evolui com o usuário.

## Contexto

Luna hoje é funcional como assistente TUI multimodal. A visão de longo prazo transcende a interface: queremos uma IA que seja genuinamente útil, memorável e evolutiva.

## Norte Estelar

### Manifesto
Luna não é apenas uma interface para APIs. É uma entidade com:
1. Personalidade consistente e evolutiva
2. Memória afetiva de interações
3. Autonomia para sugerir ações proativas
4. Estética que reflete identidade técnica gótica

### Filosofia
- Respeitar a privacidade do usuário (dados locais)
- Ser tecnicamente competente sem arrogância
- Manter a identidade gótica sem ser caricata
- Evoluir com o usuário, não contra ele

## Visão de Longo Prazo

### Assistente Pessoal Completa

#### Capacidades Expandidas
1. **Gestão de Tarefas:** Integração com todo/task managers
2. **Automação de Workflows:** Scripts e rotinas personalizadas
3. **Análise de Produtividade:** Insights sobre padrões de trabalho
4. **Pesquisa Contextual:** Web search integrado com memória

#### Inteligência Emocional
1. Reconhecimento de estado emocional via texto/voz
2. Respostas adaptadas ao contexto emocional
3. Memória de preferências e aversões do usuário
4. Sugestões proativas baseadas em histórico

#### Multimodalidade Avançada
1. Processamento de vídeo (análise de screencasts)
2. OCR integrado para documentos
3. Transcrição em tempo real de reuniões
4. Síntese de informações de múltiplas fontes

### Arquitetura Modular

#### Sistema de Plugins
1. API pública para extensões
2. Marketplace de plugins comunitários
3. Sandboxing para segurança
4. Hot-reload de módulos

#### Integração Desktop
1. Hotkeys globais para ativação
2. Overlay transparente sobre aplicações
3. Captura de contexto de janelas ativas
4. Automação de GUI via acessibilidade

### Estética e UX

#### Interface Evoluída
1. Temas customizáveis (mantendo identidade dark)
2. Animações ASCII procedurais
3. Widgets interativos (gráficos, barras, timelines)
4. Modo de apresentação para demos

#### Personalização
1. Ajuste de personalidade (intensidade do sarcasmo)
2. Preferências de verbosidade
3. Filtros de conteúdo customizáveis
4. Atalhos e aliases definidos pelo usuário

## Metas por Fase

### Q1 2026: Estabilização
1. Refatorar threading para robustez
2. Implementar testes unitários completos
3. Documentação técnica exaustiva
4. Onboarding interativo para novos usuários

### Q2 2026: Expansão Multimodal
1. Suporte a vídeo (frames + áudio)
2. OCR para screenshots e PDFs
3. Memória vetorial com embeddings
4. Busca semântica em histórico

### Q3 2026: Autonomia
1. Sistema de tarefas agendadas
2. Sugestões proativas baseadas em contexto
3. Integração com calendários e emails
4. Workflow automation (IFTTT-like)

### Q4 2026: Comunidade
1. API pública de plugins
2. Documentação de desenvolvimento
3. CLI para criação de extensões
4. Galeria de plugins oficiais

### 2027+: Além da Interface
1. Versão mobile (Termux/iOS Shortcuts)
2. Servidor local com WebUI opcional
3. Sincronização entre dispositivos
4. Modo colaborativo (múltiplos usuários)

## Princípios Norteadores

### Técnicos
1. Privacidade primeiro (dados locais, criptografia)
2. Performance (threading, cache, lazy loading)
3. Modularidade (low coupling, high cohesion)
4. Testabilidade (mocks, fixtures, CI/CD)

### Produto
1. Utilidade real sobre features vistosas
2. Curva de aprendizado suave
3. Documentação como código (sempre atualizada)
4. Feedback loops curtos (logs, métricas, crashes)

### Comunidade
1. Open source desde o início (GPLv3)
2. Contribuições bem-vindas (guias claros)
3. Comunicação transparente (roadmap público)
4. Reconhecimento de contribuidores
